## 1. Аналіз домену

У системі можна виділити кілька основних піддоменів (bounded contexts), що відповідають за різні функціональні можливості:

**User Management**
Відповідає за роботу з користувачами: реєстрацію, авторизацію та збереження стану входу у сесію.

Реалізовано через клас User та маршрути /register, /login, /logout.

Користувач може мати роль адміністратора (is_admin=True) або звичайного користувача (is_admin=False).

Стан входу зберігається у session, де тримаються user_id і is_admin.

**Review System**
Відповідає за додавання та перегляд відгуків.

Основна сутність — Review, яка містить id, title, content та user_id (foreign key на User).

Маршрути: /add_review для додавання відгуку, /reviews для перегляду всіх відгуків.

Кожен відгук прив’язаний до автора через властивість author.

**Admin Moderation**
Відповідає за модерацію, тобто видалення відгуків із правами адміністратора.

Адміністративна поведінка реалізована всередині функції /delete_review/<id>, де перевіряється, чи є користувач адміном (is_admin).

В системі немає окремої адмін-панелі, адмінські дії інтегровані у загальний функціонал модерації.

## 2. Сутності (Entities)

**User**

Містить поля: id, username, password, is_admin.

Прямий зв’язок із Review через список reviews.

Ролі: звичайний користувач (is_admin=False) або адміністратор (is_admin=True).

**Review**

Містить поля: id, title, content, user_id (foreign key на User).

Прив’язаний до автора через властивість author.

## 3. Domain Behavior (поведінка домену)
**Дія** :	**Відповідальний код**
Реєстрація   :	register()
Вхід / вихід    :	login(), logout()
Додавання відгуку   :	add_review()
Перегляд відгуків   :	reviews()
Видалення з ролями  :	delete_review()
