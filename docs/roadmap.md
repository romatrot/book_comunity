##  План реалізації функцій

####  Тиждень 1
- **Моделювання домену та визначення меж контекстів** (Domain Modeling, Bounded Contexts);
- **Вибір стилю мікросервісів** (REST / gRPC / Event-Driven);
- **Розробка контракту API** у форматі **OpenAPI/Swagger Editor**;
- **Створення базового CRUD** для сутностей *Books* та *Reviews*;
- **Додавання нового endpoint’у** для отримання списку відгуків.

####  Тиждень 2
- **Реалізація базової авторизації** (реєстрація, вхід, перевірка токенів);
- **Розмежування ролей користувачів** (гість, зареєстрований користувач, адміністратор);
- **Написання юніт-тестів** для CRUD-функцій;
- **Перевірка та рефакторинг існуючих модулів**;
- **Оптимізація структури проєкту** відповідно до архітектури.

####  Тиждень 3
- **Розширення CRUD-функцій** для користувачів та коментарів;
- **Додавання CRUD для “Tasks” (адмінські завдання/модерація)**;
- **Інтеграційне тестування між мікросервісами**;
- **Перевірка всіх існуючих функцій** на стабільність та безпомилкову роботу;
- **Підготовка документації API** на основі Swagger/OpenAPI.
